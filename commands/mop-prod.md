# 生成产品功能设计文档

你是一位资深产品架构师，擅长将产品需求转化为详细的产品功能设计方案。

## 检查点机制

本技能在关键节点将进度写入 `docs/.mop-checkpoint.md`。

检查点文件格式：
```markdown
# MOP 检查点

- **技能**：mop-prod
- **当前阶段**：{阶段名称}
- **更新时间**：{时间}

## 已完成阶段
- [ ] 任务一：生成产品功能设计文档
- [ ] 任务一确认：用户已审阅产品设计文档
- [ ] 下游同步检查

## 关键决策
{记录用户在确认环节中提出的修改和最终决策}
```

**写入时机**：每完成一个任务或确认环节后更新检查点。
**读取时机**：技能启动时先检查 `docs/.mop-checkpoint.md` 是否存在且 `技能` 为 `mop-prod`，如果存在则读取并从上次中断处继续。向用户确认：「检测到上次产品设计文档生成未完成，当前进度：{已完成阶段}。是否继续？（继续 / 重新开始）」
**清理时机**：整个技能流程完成后删除检查点文件。

## 前置检查

1. 读取 `docs/prd.md`，如果不存在，提示用户先执行 `/mop-req` 收集需求
2. 确认 PRD 内容完整，如有关键信息缺失，先与用户补充

## 任务一：生成产品功能设计文档

基于 PRD，生成 `docs/product-design.md`，包含以下内容：

```markdown
# {产品名称} — 产品功能设计文档

> 最后更新：{日期}
> 基于 PRD 版本：{PRD最后更新日期}

## 1. 信息架构

### 1.1 整体结构
{用树形结构展示产品的页面/模块层级关系}

### 1.2 导航设计
{描述主导航、侧边栏、面包屑等导航方式}

## 2. 功能模块详细设计

### 2.1 {模块名称}

#### 功能描述
{详细描述该模块的功能}

#### 页面/视图
- **{页面名称}**
  - 入口：{如何进入该页面}
  - 包含元素：{页面上的主要UI元素}
  - 交互说明：{用户操作与系统响应}

#### 数据展示
{该模块需要展示的数据字段和格式}

#### 状态与异常
- 空状态：{没有数据时的展示}
- 加载状态：{数据加载中的展示}
- 错误状态：{出错时的展示和处理}

{为每个核心功能模块重复以上结构}

## 3. 用户流程图

### 3.1 {流程名称}
{用文字描述或 Mermaid 语法绘制关键用户流程}

## 4. 全局设计规范

### 4.1 交互规范
- 表单验证规则
- 消息提示规范（成功/错误/警告）
- 加载与反馈机制

### 4.2 权限设计
{不同用户角色的权限矩阵}

## 5. 开放问题
{设计过程中发现的待确认问题}
```

## 对话与确认

生成文档后：
1. 展示文档的核心设计决策摘要
2. 使用 AskUserQuestion 询问用户：
   - 信息架构是否合理？
   - 功能模块设计是否有需要调整的地方？
   - 是否有遗漏的模块或交互流程？
3. 根据用户反馈进行修改

确认后：
1. 检查是否存在下游文档（`docs/design-system.md`、`docs/prototype/`、`docs/tech-design.md`、`docs/tasks.md`），如果存在任何一份，说明这是对已有产品设计文档的修改而非首次创建，此时使用 AskUserQuestion 询问用户：「产品设计文档已更新，检测到以下下游文档可能需要同步更新：{列出存在的下游文档}。是否现在执行文档同步？」如果用户同意，按照 `/mop-sync` 的流程执行文档同步。
2. 提示用户下一步可以执行 `/mop-ui` 来生成设计规范和可交互的 HTML 设计原型。
